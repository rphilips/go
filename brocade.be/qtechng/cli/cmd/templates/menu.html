Working with local files
Checkin
🛈 Checkin local files.

The specific action is determined by the given arguments:

  • With no arguments, all files - belonging to QtechNG - in the current working directory and its subdirectories are checked in
  • With arguments, they are considered files or directories. The files - belonging to QtechNG - are checked in. If the argument is a directory, all its files (subdirectories included) are checked in.
Properties
🛈 Get the properties of a local QtechNG file.

This menu entry has two functionalities:

  • The properties of the given file are shown
  • You have the possibility to store certain properties in the clipboard
The entry works best with an argument, a filename.
Working with files in the QtechNG repository
Search
🛈 Search for files in the QtechNG repository.

You can modify the search by specifying a few options:

  • Use lowercase if you want to transform the text before actual matching
  • regexp allows for very powerful searches but is somewhat more cumbersome
  • per line works line per line. This works best together with regexp
  • clear empties the resulting local directories on checkout

These files can be checked out as well. You have the option to specify an Edit list for convenient editing.

Checkout
🛈 Checkout sources from the QtechNG respository.

checkout works in two modes:

  • Without arguments, all files in the current directory are checked out.
  • With arguments, only the specified files are checked out
